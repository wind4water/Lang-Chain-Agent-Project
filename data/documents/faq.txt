常见问题解答 (FAQ)

========================================
一般问题
========================================

Q: 这个系统是做什么的？
A: LangChain 对话 Agent 是一个智能对话系统，可以回答问题、调用工具、查询知识库。适用于客服、技术支持、知识管理等场景。

Q: 需要什么技术背景才能使用？
A: 基础使用不需要技术背景，只需会发送 HTTP 请求。如需部署和定制，建议了解 Python 和 Web 开发基础。

Q: 支持哪些语言？
A: 系统支持中文和英文。底层的 GPT 模型支持多种语言，但界面和文档主要是中文。

Q: 可以免费使用吗？
A: 系统本身开源免费，但需要：
   - OpenAI API 账号（收费）
   - 服务器资源（如果部署到云端）
   - 可选：Langfuse 监控（有免费额度）

========================================
功能相关
========================================

Q: 对话能记住上下文吗？
A: 可以。系统会自动维护对话历史，每个 session_id 独立管理。可以通过配置上下文压缩策略来优化长对话。

Q: 支持哪些工具？
A: 内置 9 个工具：
   1. 获取当前日期和时间
   2. 获取 Unix 时间戳
   3. 获取星期几
   4. DuckDuckGo 网络搜索
   5. 网页内容抓取
   6. 生成随机数
   7. 天气查询
   8. 货币转换
   9. 计算器

Q: 可以添加自定义工具吗？
A: 可以。在 app/tools/builtin.py 中定义新工具，然后注册到工具列表。工具必须是标准的 Python 函数，并使用 @tool 装饰器。

Q: RAG 知识库支持哪些文档？
A: 目前支持：
   - 纯文本文件 (.txt)
   - Markdown 文件 (.md)
   - PDF 文件 (.pdf)
   未来将支持 Word、Excel、PPT 等更多格式。

Q: 知识库可以有多大？
A: 理论上无限制，但实际受限于：
   - 磁盘空间（向量数据库会占用空间）
   - 检索性能（文档越多，检索越慢）
   - API 成本（embedding 需要调用 API）
   建议单个知识库不超过 10000 个文档。

Q: 如何更新知识库？
A: 两种方式：
   1. 自动更新：设置 RAG_REBUILD_ON_STARTUP=true，每次启动时重建
   2. 手动更新：调用 POST /rag/rebuild 接口

========================================
技术问题
========================================

Q: 需要什么硬件配置？
A: 最低配置：
   - CPU: 2核心
   - 内存: 4GB
   - 硬盘: 10GB
   推荐配置：
   - CPU: 4核心
   - 内存: 8GB
   - 硬盘: 50GB（如果有大量文档）

Q: 支持哪些操作系统？
A: 跨平台支持：
   - Linux（推荐 Ubuntu 20.04+）
   - macOS（推荐 10.15+）
   - Windows（推荐 Windows 10+）

Q: 可以使用其他 LLM 模型吗？
A: 可以。系统基于 LangChain，理论上支持所有兼容 OpenAI API 的模型，包括：
   - OpenAI 官方模型（GPT-4, GPT-3.5 等）
   - Azure OpenAI
   - 国内大模型（智谱、百川、讯飞等，如果提供兼容接口）
   - 本地部署的模型（如 Ollama）

Q: 数据存储在哪里？
A: 默认使用 SQLite，数据存储在项目目录的 data/ 文件夹：
   - data/checkpoints.db - 对话历史
   - data/chroma/ - 向量数据库
   可以切换到 PostgreSQL 用于生产环境。

Q: 如何备份数据？
A: 备份以下文件和目录：
   - data/checkpoints.db（对话历史）
   - data/chroma/（知识库向量）
   - data/documents/（原始文档）
   - .env（配置文件，注意敏感信息）

Q: 并发能力如何？
A: SQLite 版本适合低并发场景（<10 并发）。高并发建议：
   - 切换到 PostgreSQL
   - 使用 Gunicorn/uWSGI 多进程部署
   - 配置负载均衡

Q: 响应速度慢怎么办？
A: 优化建议：
   1. 使用更快的模型（gpt-3.5-turbo）
   2. 减少 RAG 检索数量（RAG_TOP_K）
   3. 启用上下文压缩
   4. 使用更快的 embedding 模型
   5. 优化网络（使用国内 API 中转）

========================================
部署相关
========================================

Q: 如何部署到生产环境？
A: 生产部署检查清单：
   1. 使用 PostgreSQL 替代 SQLite
   2. 配置反向代理（Nginx）
   3. 启用 HTTPS
   4. 设置 API Key 认证
   5. 配置日志和监控
   6. 使用 Supervisor/systemd 管理进程
   7. 设置自动备份
   8. 配置防火墙和安全组

Q: 支持 Docker 部署吗？
A: 目前项目未提供官方 Dockerfile，但可以自行编写。参考配置：
   - 基础镜像：python:3.10-slim
   - 暴露端口：8000
   - 挂载卷：data/、.env

Q: 可以部署到云平台吗？
A: 可以，支持主流云平台：
   - 阿里云（ECS、函数计算）
   - 腾讯云（CVM、云函数）
   - AWS（EC2、Lambda）
   - Azure（VM、Functions）
   - Vercel、Railway 等 PaaS 平台

Q: 如何配置域名和 HTTPS？
A: 使用 Nginx 反向代理：
   1. 申请 SSL 证书（Let's Encrypt 免费）
   2. 配置 Nginx 监听 443 端口
   3. 代理请求到本地 8000 端口
   4. 配置自动续期

========================================
成本相关
========================================

Q: 使用成本是多少？
A: 主要成本来自：
   1. OpenAI API 调用
      - gpt-4o-mini: $0.15/1M tokens（输入）+ $0.6/1M tokens（输出）
      - embedding: $0.02/1M tokens
   2. 服务器成本（如果云部署）
      - 最低 2核4G 约 ¥100/月
   3. Langfuse 监控（可选）
      - 免费版 5000 traces/月
      - Pro 版 $59/月起

Q: 如何降低成本？
A: 节约技巧：
   1. 使用便宜模型（gpt-3.5-turbo）
   2. 启用上下文压缩
   3. 降低 RAG 检索数量
   4. 缓存常见问题答案
   5. 设置 Langfuse 采样率 <1.0
   6. 使用国内模型替代 OpenAI

Q: 可以按月付费吗？
A: OpenAI API 是按量付费，用多少付多少。建议设置用量警报和月度预算上限。

========================================
安全与隐私
========================================

Q: 数据安全吗？
A: 安全措施：
   - 对话历史仅存储在本地数据库
   - 不会主动上传到第三方（除 OpenAI API）
   - 支持自托管，完全控制数据
   - 建议：启用数据库加密、定期备份

Q: OpenAI 会看到我的数据吗？
A: OpenAI API 会接收您的输入和输出，根据其隐私政策：
   - API 数据不用于训练模型（自 2023 年 3 月起）
   - 数据保留 30 天用于滥用检测
   - 可选择加入零保留计划

Q: 支持数据加密吗？
A: 可以：
   - 数据库加密：使用 SQLCipher 加密 SQLite
   - 传输加密：启用 HTTPS
   - 存储加密：使用文件系统级加密

Q: 如何防止滥用？
A: 防护措施：
   1. API Key 认证
   2. 速率限制
   3. IP 白名单
   4. 输入内容过滤
   5. 监控异常使用

========================================
故障排除
========================================

Q: 服务启动失败怎么办？
A: 检查步骤：
   1. 确认 Python 版本（需要 3.10+）
   2. 确认依赖安装完整（pip list）
   3. 检查 .env 配置（API Key 等）
   4. 查看错误日志
   5. 确认端口 8000 未被占用

Q: RAG 查询总是说"无法回答"？
A: 可能原因：
   1. 知识库为空（检查 data/documents/）
   2. 文档未重建索引（调用 /rag/rebuild）
   3. 相似度阈值过高（降低 RAG_SCORE_THRESHOLD）
   4. 问题与文档内容不匹配

Q: Token 统计不准确？
A: Token 统计基于 tiktoken 估算，可能与实际略有差异（±5%）。以 OpenAI 账单为准。

Q: Langfuse 显示错误？
A: 已知问题：Langfuse 与 LangGraph 存在兼容性问题，可能显示 "NoneType" 或 "run not found" 错误。这些错误已被安全包装器捕获，不影响实际功能，可以忽略。
